# Руководство по контрибуции

Спасибо, что решили помочь улучшить проект! 🙌

## Как внести вклад

### 1. Сообщите об ошибке
- Проверьте, не существует ли уже похожего issue
- Создайте новый issue с описанием:
  - Шаги воспроизведения
  - Ожидаемое поведение
  - Фактическое поведение
  - Скриншоты (если возможно)
  - Версия Python и ОС

### 2. Предложите улучшение
- Создайте issue с меткой "enhancement"
- Опишите, что вы хотите улучшить и почему

### 3. Pull Request
- Форкните репозиторий
- Создайте ветку: `feature/название-фичи` или `fix/название-бага`
- Внесите изменения
- Напишите тесты (если возможно)
- Обновите документацию
- Откройте Pull Request

## Стиль кода

- Следуйте PEP 8
- Используйте понятные имена переменных
- Добавляйте docstrings для функций и классов
- Комментируйте сложные участки кода

## Тестирование

Перед отправкой PR убедитесь, что:
- Код работает без ошибок
- Не сломана существующая функциональность
- Добавлены тесты для новой функциональности (если применимо)
